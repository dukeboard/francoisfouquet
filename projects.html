Kevoree

Build around the Model@Runtime techniques following the Diva project, this project aims at taming the complexity of building distributed adaptive systems. Kevoree modeling tools offer an abstract ways to continuously design and develop components based distributed systems. This model allows to express most of distributed systems concerns like Node, Components, Communication channels and synchronization groups. By distributing architectural models itself Kevoree allows to manage cluster of nodes with severals algorithms like Gossip, Paxos. Initially build for Java platform, Kevoree runtime now cover and manages heterogeneous devices like:

  • IoT devices on micro-controllers (Arduino, AVR, ARM)
  • Android platforms (Phone, Tablet)
  • Cloud infrastructures (KVM, EC2, FreeBSD Jails)

Role:

Project initial architect and main developer

Technologies:

Scala, Java, C, Processing, EMF, Arduino, AVR, OSGi, Maven

Link:

Kevoree.org

Target case study:

  • Home automation gateways
  • Tactical on-field systems for firefighters
  • SmartBuilding
  • Cloud PaaS
  • Sensor networks

Kevoree Modeling Frameworks

This project was developed to improve the use of modeling techniques@Runtime. KMF offers an alternative to Eclipse Modeling Framework by provided generator from EMOF model to Java plain POJO. Compliance with XMI format this framework outperform EMF on memory usage and load/save mecanisme allowing to be embedded in tiny platforms.

Role:

Project co-initial architect and developer

Technologies:

Scala, Java, EMF

Link:

KMF GitHub
